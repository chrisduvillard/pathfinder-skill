# Code of Conduct

## Our pledge

We want Pathfinder to be a welcoming, constructive project for people improving
agent workflows, repository discovery, and open-source automation.

Participants are expected to be respectful, patient, and clear. Disagreement is
fine. Harassment, threats, personal attacks, sustained disruption, or publishing
private information is not.

## Expected behavior

- Use welcoming and inclusive language.
- Assume good intent while still being precise about technical risk.
- Keep criticism focused on code, docs, workflows, and behavior.
- Respect maintainer time by keeping issues and pull requests scoped.
- Report security issues privately through `SECURITY.md`.

## Unacceptable behavior

- Harassment, insults, threats, or discriminatory language.
- Sexualized language or unwanted attention.
- Publishing private messages, personal data, credentials, or private repository
  material without consent.
- Trolling, repeated off-topic comments, or deliberate disruption.
- Pressuring maintainers to bypass security or review processes.

## Enforcement

Maintainers may remove comments, close issues, block users, or restrict
participation when behavior harms the project or community.

Reports can be made privately through GitHub's repository contact mechanisms.
Security vulnerabilities should follow `SECURITY.md` instead of this process.

This policy follows the spirit and structure of the Contributor Covenant while
using wording tailored to this repository.
